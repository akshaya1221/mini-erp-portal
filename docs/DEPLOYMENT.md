# Deployment Blueprint Reference

This document outlines the hosting architecture and env properties required to host the **Mini ERP + CRM Operations Portal** live.

---

## 1. Hosting Architecture

The portal is separated into two deployment segments:
1.  **Backend Services**: Render Web Service running Node.js + Express (linked to Neon Postgres database).
2.  **Client Application**: Vercel Static Hosting running the Vite + React client bundle.

---

## 2. One-Click Blueprint Deployment (Render)

A `render.yaml` blueprint configuration file is included in the project root. To deploy your entire backend and database infrastructure:
1. Log in to the [Render Dashboard](https://dashboard.render.com/).
2. Navigate to **Blueprints**.
3. Connect your GitHub repository and follow the configuration steps. Render will automatically parse `render.yaml` and provision your services.

Alternatively, you can configure the service manually.

---

## 3. Manual Backend Deployment (Render)

*   **Service Type**: Web Service
*   **Runtime**: `Node`
*   **Root Directory**: `backend`
*   **Build Command**: `npm install && npm run prisma:generate && npm run build`
*   **Start Command**: `npm start`
*   **Environment Variables**:
    *   `DATABASE_URL`: `postgresql://<user>:<password>@<host>/<database>?sslmode=require`
    *   `JWT_SECRET`: A secure random cryptographic string (e.g., `a-secure-32-character-secret-key`)
    *   `PORT`: `4000`

---

## 4. Manual Frontend Deployment (Vercel)

*   **Build Preset**: `Vite` (Vercel will auto-detect the bundler)
*   **Root Directory**: `frontend`
*   **Build Command**: `npm run build`
*   **Output Directory**: `dist`
*   **Environment Variables**:
    *   `VITE_API_URL`: `https://mini-erp-backend.onrender.com` (Set this to the live backend URL generated by Render, with **no trailing slash**).

---

## 5. Routing Rewrites (Vercel Config)

To prevent Vite client-side route reloads (like `/dashboard`, `/customers`) from throwing `404 Not Found` errors in production, a `vercel.json` file is placed inside the `frontend/` root:

```json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```
This maps all path requests back to the main document entrypoint `index.html`, where **React Router DOM** resolves client views.
